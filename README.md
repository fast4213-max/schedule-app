## 構成概要

| 役割 | サービス | 費用 |
|------|----------|------|
| 画面（フロントエンド） | GitHub Pages | 無料 |
| データ保存・API | Google Apps Script + スプレッドシート | 無料 |

```
ブラウザ（index.html）
　↓ fetch POST / GET
GAS Web App（gas_code.js）
　↓ 読み書き
Google スプレッドシート
　├ responses     回答データ
　├ participants  参加者・パスワード
　└ settings      管理者設定
```

> **設定はスプレッドシートで管理されるため、どの端末・ブラウザからアクセスしても同じ設定・データが表示されます。**

## セットアップ手順

### 1. スプレッドシートを用意する

1. [Google スプレッドシート](https://sheets.google.com) にアクセス
2. 「＋ 新しいスプレッドシート」で空のファイルを作成
3. タイトルは何でもOK（例：`スケジュール調整DB`）

> シート（responses / participants / settings）は初回送信時に**自動生成**されます。手動で作る必要はありません。

---

### 2. GAS（Google Apps Script）を設定する

1. 作成したスプレッドシートを開く
2. メニュー「拡張機能」→「Apps Script」をクリック
3. エディタ内のコードを**全削除**して `gas_code.js` の内容を貼り付ける
4. `Ctrl+S`（Mac: `⌘S`）で保存

#### デプロイ（公開）する

1. 右上「デプロイ」→「新しいデプロイ」
2. 以下のように設定する

| 項目 | 設定値 |
|------|--------|
| 種類 | ウェブアプリ |
| 次のユーザーとして実行 | 自分 |
| アクセスできるユーザー | **全員（匿名ユーザーを含む）** ← 重要 |

3. 「デプロイ」→ Googleアカウントの権限確認が出たら「許可」
4. 表示された **ウェブアプリのURL** をコピーして控える

```
例: https://script.google.com/macros/s/xxxxxxxxxxxxxxxx/exec
```

> GASのコードを修正したあとは「デプロイ」→「デプロイを管理」→「✏ 編集」→「バージョン：新しいバージョン」→「デプロイ」で更新が必要です。

---

### 3. index.html を編集する

`index.html` の先頭付近にある以下の行を見つけて、控えたURLに書き換えます。

**変更前**
```js
const GAS_URL = 'YOUR_GAS_URL_HERE';
```

**変更後**
```js
const GAS_URL = 'https://script.google.com/macros/s/xxxxxxxxxxxxxxxx/exec';
```

---

### 4. GitHub Pages に公開する

#### リポジトリを作成する

1. [GitHub](https://github.com) にログイン（アカウントがなければ無料登録）
2. 右上「＋」→「New repository」
3. 以下のように設定する

| 項目 | 設定値 |
|------|--------|
| Repository name | `schedule-app`（任意） |
| 公開設定 | **Public**（無料プランでPages利用に必要） |
| Add a README file | チェックを入れる |

4. 「Create repository」をクリック

#### index.html をアップロードする

1. 作成したリポジトリを開く
2. 「Add file」→「Upload files」
3. `index.html` をドラッグ＆ドロップ
4. 「Commit changes」をクリック

#### GitHub Pages を有効にする

1. リポジトリの「Settings」タブ
2. 左メニュー「Pages」
3. Branch: `main` / フォルダ: `/(root)` を選択
4. 「Save」をクリック

1〜2分後に以下のURLでアクセスできます。

```
https://あなたのユーザー名.github.io/schedule-app/
```

## 管理者設定

画面右上の「⚙ 管理」ボタンから管理者パネルを開きます。

**初期パスワードは `0000` です。最初にログイン後、必ず変更してください。**

### 設定項目

| 項目 | 説明 |
|------|------|
| イベント名 | アプリのタイトルに表示される名前 |
| 管理者パスワード変更 | 4桁の数字で新しいパスワードを設定。スプレッドシートに反映される |
| 対象月 | カレンダーの表示範囲（1〜2ヶ月）を設定。同じ月にすると1ヶ月表示 |
| 回答を締め切る | ONにすると回答フォームが非表示になる |

> 設定はスプレッドシートの `settings` シートに保存されます。どの端末からアクセスしても設定は同じです。

### データ削除

| ボタン | 動作 |
|--------|------|
| 特定の参加者を削除 | 名前を選択して削除。確認モーダルが1回表示される |
| 全員のデータをクリア | 全参加者の回答を削除。確認モーダルが**2回**表示される |

## 利用者の使い方

### 回答する

1. 「回答する」タブを開く
2. 名前を入力する
3. 4桁のパスワードを設定する（修正時に使うので覚えておく）
4. カレンダーで日付をタップ → ○ △ ✕ が切り替わる
5. 「回答を送信」をタップする

| 記号 | 意味 |
|------|------|
| ○ | 参加OK |
| △ | たぶん参加できる |
| ✕ | 参加不可 |
| （空白） | タップするたびに ○ → △ → ✕ → 空白 と切り替わる |

---

### 回答を修正する

1. 「ダッシュボード」タブを開く
2. 参加者一覧から自分の名前をタップする
3. 「回答する」タブに切り替わり、自分の回答がカレンダーに表示される
4. 「✏ この内容を修正する」をタップする
5. 4桁のパスワードを入力して「確認」
6. カレンダーを編集して「修正を保存」をタップする

---

### ダッシュボードを見る

- 日付ごとに ○ / △ / ✕ の**人数**が表示される
- 人数をタップすると**名前一覧**のポップアップが開く
- 月ナビ「‹ ›」で前後の月を切り替えられる（対象月の範囲内）
- 参加者一覧の名前をタップするとその人の回答をカレンダーで確認できる

## スプレッドシートの構造

### responses シート（回答データ）

| 列 | 内容 | 例 |
|----|------|----|
| 名前 | 回答者名 | 田中 |
| 日付 | YYYY-MM-DD形式 | 2026-06-21 |
| 回答 | ok / maybe / ng | ok |
| pin_hash | パスワードのハッシュ値（自動） | 168c00 |
| 更新日時 | ISO 8601形式（自動） | 2026-06-09T10:23:00.000Z |

> 1人が複数日を選ぶと行が複数になります（1行 = 1日付）。修正時は一度全行削除して再登録します。

### participants シート（参加者）

| 列 | 内容 |
|----|------|
| 名前 | 参加者名 |
| pin_hash | パスワードのハッシュ値 |
| 登録日時 | 初回回答日時 |

### settings シート（管理者設定）

| key | value の内容 |
|-----|-------------|
| eventName | イベント名 |
| adminPin | 管理者パスワードのハッシュ値 |
| closed | 締め切り状態（true / false） |
| month1 | 対象月の開始（JSON形式） |
| month2 | 対象月の終了（JSON形式） |

## トラブルシューティング

| 症状 | 確認・対処 |
|------|-----------|
| 送信してもデータが保存されない | `GAS_URL` が正しく設定されているか確認。GASのデプロイで「全員アクセス可」になっているか確認 |
| GASデプロイ後にエラーになる | コードを修正した場合は「新しいバージョン」として再デプロイが必要 |
| GitHub Pagesが表示されない | Save後1〜2分待つ。リポジトリが Public になっているか確認 |
| 管理者パスワードを忘れた | スプレッドシートの `settings` シートを開き、`adminPin` の行を削除する。次回ログイン時に初期値（`0000`のハッシュ）に戻る |
| 別の端末で設定が違う | GAS_URLが同じファイルを参照していれば設定は一致するはず。ブラウザキャッシュをクリアして再読み込みを試す |
| パスワードを忘れて修正できない | スプレッドシートの `participants` シートで該当者の `pin_hash` 行を削除 → 管理者が該当者のデータを削除して再回答してもらう |

---

*このアプリは GitHub Pages（静的ホスティング）と Google Apps Script（サーバーレス API）のみで動作します。サーバー費用は一切かかりません。*
